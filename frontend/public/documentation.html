REPO Research Monitoring System Documentation

Use this page for reading, printing, or downloading a Word-openable file.

1. System Overview

This project is a full-stack Research Extension Publication Office (REPO) Research Monitoring System for Benguet State University.

It is used to:

  • Manage publication records
  • Track faculty research metrics (citations, H-index, i10-index)
  • Match publication authors to faculty profiles
  • Generate dashboard KPIs and trend analytics
  • Manage acknowledgements records
  • Manage users and login sessions
  • Record audit logs for key actions

2. Technology Stack

  • Frontend: Vue 3 + Vite + Tailwind + Axios + Chart.js + ExcelJS
  • Backend: PHP 8.1+ + CodeIgniter 4 (REST APIs)
  • Database: MySQL (MySQLi driver)
  • Auth: Session-based authentication with role checks

3. Repository Structure

  • frontend/: Vue SPA
  • backend/: CodeIgniter API and DB migrations

Key backend folders:

  • backend/app/Controllers/Api: REST controllers
  • backend/app/Models: DB models
  • backend/app/Database/Migrations: schema definitions
  • backend/app/Libraries/AuditLogger.php: audit helper

Key frontend folders:

  • frontend/src/views: page-level modules (Dashboard, Publications, Faculty, etc.)
  • frontend/src/composables/useAuth.js: auth state helper
  • frontend/src/main.js: active route definitions and route guards

4. Core Modules

  • Dashboard: KPIs, yearly trends, college/type analytics, activity snapshots
  • Publications: CRUD, filtering, pagination, bulk import, duplicate detection
  • Faculty: CRUD, filtering (including masterlist-linked fields), top rankings
  • Author Matches: pending/confirmed publication-to-faculty links
  • Acknowledgements: CRUD with item lines (volumes/copies/remarks)
  • Audit Logs: paginated action log with search
  • Users: admin-managed user creation/status updates

6. API Base and CORS

  • Frontend API base: VITE_API_URL (fallback: http://localhost:8080/api)
  • Backend allowed frontend origin: FRONTEND_ORIGIN (fallback: http://localhost:5173)
  • Credentials are enabled (withCredentials=true) for session cookies.

7. Authentication and Authorization

Auth endpoints:

  • POST /api/auth/login
  • GET /api/auth/me
  • POST /api/auth/logout

Behavior:

  • Session stores: user_id, username, role, login timestamps
  • Login throttling: lockout for 60 seconds after 5 failed attempts
  • Frontend route guard checks /auth/me for protected pages
  • Role model: guest/viewer should be view-only, editor can add/edit records, admin can also manage users
  • UsersController currently enforces admin for user create/list/update
  • Recommendation: enforce write permissions server-side for publications/faculty/acknowledgements to fully protect view-only users

9. Database Schema (Main Tables)

  • publications
  • faculty
  • publication_author_links
  • faculty_masterlist
  • acknowledgements
  • acknowledgement_items
  • users
  • audit_logs
  • tracking_log

Highlights:

  • faculty uses soft deletes (deleted_at)
  • publication_author_links stores match status (pending, confirmed, rejected)
  • users.username is unique
  • acknowledgement_items.acknowledgement_id has FK cascade to acknowledgements

10. Local Development Setup

Backend

  1. cd backend
  2. composer install
  3. Configure backend/.env
  4. php spark migrate
  5. php spark serve --host localhost --port 8080

Current DB defaults in backend/.env:

  • Host: localhost
  • Database: remis_db
  • User: root
  • Password: empty
  • Port: 3306

Frontend

  1. cd frontend
  2. npm install
  3. Optionally set VITE_API_URL=http://localhost:8080/api
  4. npm run dev

11. Current Implementation Notes

  • frontend/src/router/index.js is template/default and not used by the running app. Active routes are in frontend/src/main.js.
  • FacultyMasterlistController exists with CRUD/import/export logic, but its endpoints are not currently registered in backend/app/Config/Routes.php.
  • Some controllers set CORS headers in constructors while a CORS filter class also exists.
  • Dashboard yearly range is currently hardcoded to 2017-2025 in DashboardController.
